O que é MySQL?

  •   Um sistema de gestão de base de dados relacional, open-source e extremamente popular.
  •   Funciona como uma "biblioteca digital" super organizada para guardar e aceder a dados.
  •   Utiliza a linguagem SQL, o padrão universal para comunicar com bases de dados.
  •   É a espinha dorsal de inúmeras aplicações web, desde startups a gigantes da tecnologia.

O que é Análise de Dados?

  •   A arte de inspecionar, limpar e modelar dados para descobrir informações úteis.
  •   O seu principal objetivo é apoiar a tomada de decisões com base em evidências.
  •   É como ser um detetive que encontra padrões e histórias escondidas nos números.
  •   Bases de dados como o MySQL são o ponto de partida, fornecendo a matéria-prima para a análise.

Setup do Ambiente

Vamos preparar o seu computador com as ferramentas essenciais para começar a trabalhar com MySQL.

Passo 1: Instalar o MySQL Server

O MySQL Community Server é o motor da sua base de dados. É aqui que a magia acontece.

  •   Aceda à página oficial e procure por "MySQL Community (GPL) Downloads".
  •   Escolha o "MySQL Installer for Windows" para uma instalação simplificada.
  •   No "Setup Type", selecione Developer Default para incluir todas as ferramentas.
  •   Crie uma senha segura para o utilizador 'root' e guarde-a bem!
Baixar MySQL Server

Passo 2: Usar o MySQL Workbench

O Workbench é a sua interface gráfica (GUI). É o seu cockpit para controlar a base de dados de forma visual e intuitiva.

  •   Se escolheu "Developer Default", o Workbench já está instalado.
  •   Se não, pode baixá-lo separadamente no site oficial.
  •   Abra o programa, clique na conexão local e insira a sua senha de 'root'.
  •   Pronto! Está conectado e pronto para criar.
Baixar MySQL Workbench

Comandos Essenciais de SQL

Domine os comandos fundamentais com exemplos genéricos que pode adaptar para qualquer projeto.

Criar um Banco de Dados

O ponto de partida. Cria o "contêiner" para as suas tabelas.

CREATE DATABASE seu_banco_de_dados;

Criar uma Tabela

Define a estrutura de uma tabela, colunas, tipos de dados e regras.

CREATE TABLE sua_tabela (
    id INT NOT NULL PRIMARY KEY,
    nome VARCHAR(100) NOT NULL,
    data_cadastro DATE
);

Adicionar Chave Estrangeira

Cria uma ligação entre duas tabelas para garantir a consistência dos dados.

ALTER TABLE sua_tabela_filha
ADD FOREIGN KEY (id_da_tabela_pai) REFERENCES sua_tabela_pai(id);

Realizar uma Consulta

Busca dados específicos, filtrando os resultados com a cláusula `WHERE`.

-- Seleciona todos os registos onde o nome é 'Maria'
SELECT * FROM sua_tabela WHERE nome = 'Maria';

Siga o passo a passo completo para criar e popular a sua primeira base de dados funcional.

1. Criar a Base de Dados

O comando inicial que cria o espaço para o nosso projeto.

CREATE DATABASE sn_bank;

2. Criar as Tabelas

Definir a "planta" de cada tabela, especificando todas as colunas e as suas regras.

USE sn_bank;

CREATE TABLE tabelaclientes (id_cliente VARCHAR(20) NOT NULL PRIMARY KEY, DataNascimento DATE NOT NULL, CPF VARCHAR(11) NOT NULL, Nome VARCHAR(50) NOT NULL, Email VARCHAR(50) NOT NULL, Rua VARCHAR(50) NOT NULL, Bairro VARCHAR(50) NOT NULL, Cidade VARCHAR(50) NOT NULL, Estado CHAR(2) NOT NULL, CEP CHAR(8) NOT NULL, funcional VARCHAR(20) NOT NULL);
CREATE TABLE tabelaconta (num_conta VARCHAR(20) PRIMARY KEY, TipoConta VARCHAR(50) NOT NULL, Saldo DECIMAL (10,2) NOT NULL, DataAbertura DATE);
CREATE TABLE tabelaclienteconta (num_conta VARCHAR(20), id_cliente VARCHAR(20), PRIMARY KEY (num_conta, id_cliente));
CREATE TABLE tabeladepartamento (id_departamento VARCHAR(20) NOT NULL PRIMARY KEY, NomeDepartamento VARCHAR(50) NOT NULL, NumeroDepartamento VARCHAR(20) NOT NULL, funcional_gerente VARCHAR(20) NOT NULL);
CREATE TABLE tabelacolaboradores (funcional VARCHAR(20) NOT NULL PRIMARY KEY, Salario DECIMAL (10,2), NomeColaborador VARCHAR(50) NOT NULL, EmailColaborador VARCHAR(50), Cargo VARCHAR(50) NOT NULL, CPFColaborador VARCHAR(11) NOT NULL, TelefoneColaborador VARCHAR(11) NOT NULL, id_departamento VARCHAR(20) NOT NULL);
CREATE TABLE tabelaemprestimos (num_contrato VARCHAR(20) PRIMARY KEY, Status BOOLEAN NOT NULL, Prazo INTEGER NOT NULL, DataInicio DATE NOT NULL, Tipo VARCHAR(20) NOT NULL, Valor DECIMAL (10,2) NOT NULL, id_cliente VARCHAR(20) NOT NULL, id_colaborador VARCHAR(20) NOT NULL);
CREATE TABLE tabelapagamento (id_pagamento VARCHAR(20) PRIMARY KEY, DataPagamento DATE NOT NULL, Valor DECIMAL (10,2) NOT NULL, Status VARCHAR(20) NOT NULL, num_contrato VARCHAR(20) NOT NULL);
CREATE TABLE tabelascorecredito (id_score VARCHAR(20) NOT NULL PRIMARY KEY, DataConsulta DATE, Fonte VARCHAR(50) NOT NULL, Pontuacao INTEGER NOT NULL, Justificativa TEXT, id_cliente VARCHAR(20) NOT NULL);
CREATE TABLE tabelatelefones (id_telefone VARCHAR(20) PRIMARY KEY, Telefone VARCHAR(11) NOT NULL, id_cliente VARCHAR(20) NOT NULL);

4. Adicionar Chaves Estrangeiras

Criar as ligações vitais entre as tabelas para garantir a integridade dos dados.

ALTER TABLE tabelaclienteconta ADD FOREIGN KEY (id_cliente) REFERENCES tabelaclientes(id_cliente), ADD FOREIGN KEY (num_conta) REFERENCES tabelaconta(num_conta);
ALTER TABLE tabelacolaboradores ADD FOREIGN KEY (id_departamento) REFERENCES tabeladepartamento(id_departamento);
ALTER TABLE tabelaemprestimos ADD FOREIGN KEY (id_cliente) REFERENCES tabelaclientes(id_cliente), ADD FOREIGN KEY (id_colaborador) REFERENCES tabelacolaboradores(funcional);

Explorar os dados para garantir que tudo está correto e começar a extrair insights.

-- Contar quantos clientes existem em São Paulo (SP)
SELECT COUNT(*) FROM tabelaclientes WHERE Estado = 'SP';

Análise de Dados com SQL

Vamos mergulhar nos dados e extrair informações valiosas com consultas mais avançadas.

Inspeção e Estrutura

Começamos por visualizar amostras dos dados e entender a estrutura de cada tabela.

-- Visualizar os primeiros 10 registos da tabela de clientes
SELECT * FROM tabelaclientes LIMIT 10;
-- Descrever a estrutura (colunas e tipos de dados) da tabela de colaboradores
DESCRIBE tabelacolaboradores;

Agregações e Contagens

Use funções como `COUNT`, `SUM`, `AVG`, `MIN` e `MAX` para resumir os dados e obter métricas importantes.

-- Contar o total de clientes
SELECT COUNT(*) AS total_clientes FROM tabelaclientes;

-- Calcular o saldo médio em todas as contas
SELECT AVG(Saldo) AS saldo_medio FROM tabelaconta;

-- Encontrar o maior salário entre os colaboradores
SELECT MAX(Salario) AS maior_salario FROM tabelacolaboradores;

Agrupamento de Dados

A cláusula `GROUP BY` agrupa linhas que têm os mesmos valores em colunas especificadas, permitindo realizar agregações em cada grupo.

-- Contar quantos clientes existem por estado
SELECT Estado, COUNT(id_cliente) AS numero_de_clientes
FROM tabelaclientes
GROUP BY Estado
ORDER BY numero_de_clientes DESC;

Manipulação de Dados

Use `INSERT`, `UPDATE` e `DELETE` para adicionar, modificar e remover registos das suas tabelas.

-- Inserir um novo cliente na tabela (exemplo)
INSERT INTO tabelaclientes (id_cliente, DataNascimento, CPF, Nome, Email, Rua, Bairro, Cidade, Estado, CEP, funcional)
VALUES ('C90001', '1990-01-01', '12345678901', 'João Silva', 'joao@email.com', 'Rua A', 'Centro', 'Cidade B', 'SP', '12345678', 'F0001');

-- Atualizar o status de um pagamento
UPDATE tabelapagamento SET Status = 'Pago' WHERE id_pagamento = 'ID_EXEMPLO';

-- Deletar um cliente
DELETE FROM tabelaclientes WHERE id_cliente = 'C90001';
